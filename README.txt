My program consists of a mandelbulb fractal renderer by means of raymarching via a compute shader. My program utilizes a compute pipeline to raymarch a 3D fractal the Mandelbulb, colors it unclamped, and stores it to a storage image. The storage image is then sent to a resolve pipeline where the storage image is sampled through a fullscreen triangle and then converted from HDR to LDR using an ACES tone mapping equation, to where the output is screen.
A user may interact with the program in the following way: The user may increase or decrease the DE/Shadow hit epsilon value to increase or decrease detail of the fractal accordingly using the up or down arrow keys, the user may change the way the fractal is colored by pressing the N key to change the way the base albedo color is calculated from either (default) normal of hit point or normal*point v1 and normal*point v2 cycling in that order, the user may press the L key to toggle the statically positioned light to be dynamically moving and orbiting the fractal in the yz-axis or to be static, and last but not least the user can explore different powers of the Mandelbulb fractal by pressing the 0-9 keys which do the following: keys 2-9 produce powers 2-9 respectively, key 1 produces power 10, and key 0 produces an iteration from power 2-10 so you can see the fractal dynamically go through states aka the "power tour". After pressing each key, the user will receive a written message in the terminal describing what the pressing that button did.
The program can be built/ran by running the following command(s) while being the project directory, "mingw32-make -C build" and then to run the program run the command "./build/FP.exe"
Things I would improve upon given more time would be the zooming in with maintained detail (ran out of time for this), adding more lights in, built in screen capture, and maybe adding more post processing such as bloom.
Pretty proud of this project as I learned a lot, and was through a lot of online help and research I was able to cleanly accomplish the current version. I used lots of sources all of which can be found inside sources/ProjectNotes.txt file.
There is a small bug for when the user chooses to do the "power tour" (pressing 0 key) where the fractal will iterate through the powers 2-10 and when in between whole power integers with the floating point value the fractal has a weird artifact that can be visible briefly, other than that no other current bugs that I am aware of. I did have a weird bug where 50% of the time the executable would properly render the fractal and the other 50% of the time would render an egg looking shape, but I believe I traced the bug to an uninitialized variable and is now fixed.
